Finance Export File Description

In order to automate the data exchange from the SOLA system to an external financial system, this export function can be used to export details from SOLA that describe information about rights, such as state leases, that have an associated obligation to make regular payments. The management of these financial obligations will be undertaken in the financial system with SOLA providing details on new rights (that lead to financial obligations) as well as when changes occur to existing rights with financial obligations.

The details exported describe:

  1. The person or organization responsible for making these payments (payer);
  2. The amount of those regular payments (e.g. ground rent);
  3. The right holders names; and
  4. Plot (parcel) area and descriptor.

    5. The export produces a CSV file. The first row in a CSV file contains field headers. The table describes every field in the sequence they appear in the export file.

    Options available to the SOLA user (with the appropriate SOLA role) initiating an export to specify the right (lease is the only option for LAA implementation of SOLA) and this search can be further qualified by specifying a date range for the date the right was registered. A list of right records meeting these criteria is displayed in a results listing.

    The SOLA user then has the option to export all records returned from the search or to select individual records. At this point, the SOLA user is prompted to identify which folder the export file will be saved in and the file name of the export file.

    In the loading of the SOLA export file into the financial system, care must be taken to ensure that where a right (a lease) has been transferred from one person to another that the record with a “current” value for the RightStatus field reflects the record containing new right holders and the new payer. Secondly, it is important that the RightTrackingNumber field together with TitleId field are used to locate the right (lease) record in the finance system when making changes to an existing right (lease) record. The RightID field should NOT be used for this purpose.

    18

    RightTrackingNumber

    Right tracking number links changes in the right history. This number is very important and plays a role of unique right identifier for a given title. With every change to the right like transfer or termination, a new copy of the current right will be created containing its own unique ID. With the tracking number you can trace and pickup previous states of the right. For example if a right gets terminated you will get a new record with unique ID and terminated state but with the same tracking number, so you can refer back to the original record with active state and cancel/terminate it in your system. This number must be used together with TitleId value to find a right for a given title.

    34

    RightStatusDate

    System date when the status of right was actually changed in the system. This date is very reliable in understanding when the right was created or changed. For example if the right has “historic” status, RightStatusDate indicates when it became historic (terminated).

    RightStatus

    Right status. Possible values:

    • current (active approved records)
    • historic (terminated records)
